# Roadmap

Milestones 1–8 establish the typed backend, model boundary, debate, MCP memory, delayed feedback,
responsive product, tracing, evaluation, and security documentation.

Milestone 9 extracts Wallet behind an A2A 1.0 remote boundary while retaining local/remote
contract parity and deterministic failure tests. Later work should prioritize authenticated
multi-user storage, deletion/export controls, production telemetry export, real provider adapters,
human-labelled quality evaluation, and broader browser accessibility coverage.

Protocols remain incremental: MCP is used for tool/context access; A2A is introduced only for an
independently deployable agent. Other committee members should be extracted only when deployment,
ownership, or interoperability requirements justify the added network lifecycle.
